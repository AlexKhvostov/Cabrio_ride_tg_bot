# Структура базы данных Cabrio Club

## Описание файлов

- `members.txt` - описание таблицы участников клуба
- `cars.txt` - описание таблицы автомобилей
- `invitations.txt` - описание таблицы приглашений новых участников
- `diagram.mmd` - диаграмма связей между таблицами (формат Mermaid)

## Связи между таблицами

1. Один участник (`members`) может иметь несколько автомобилей (`cars`)
2. Один участник (`members`) может оставить несколько приглашений (`invitations`)
3. К одному автомобилю (`cars`) может быть привязано несколько приглашений (`invitations`)

## Основные сущности

### Участники (members)
- Хранение информации об участниках клуба
- Включает личные данные, контакты и статус участника
- Обязательные поля: id, telegram_id, first_name, join_date, status

### Автомобили (cars)
- Хранение информации об автомобилях (как участников, так и потенциальных участников)
- Включает технические характеристики, статус и фотографии
- Обязательные поля: id, brand, model, year, status

### Приглашения (invitations)
- Отслеживание работы с потенциальными участниками
- Включает информацию о месте и дате приглашения, контактах и статусе
- Обязательные поля: id, car_id, invitation_date, location, inviter_member_id, status
