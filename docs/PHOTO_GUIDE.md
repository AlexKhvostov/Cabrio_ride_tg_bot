# 📸 Руководство по работе с фотографиями

## Возможности бота с фотографиями

### 🧑‍💼 Фото профиля участника
- **Когда:** Во время регистрации после заполнения информации о себе
- **Как:** Просто отправьте фотографию боту
- **Можно пропустить:** Да, командой `/skip`
- **Сохраняется:** В поле `photo_url` таблицы `members`
- **Формат файла:** `member_{telegram_id}_{timestamp}.jpg`

### 🚗 Фотографии автомобилей
- **Когда:** При добавлении автомобиля после указания года выпуска
- **Как:** Отправляйте фотографии одну за другой
- **Количество:** Неограниченно
- **Завершение:** Команда `/done` или `/skip`
- **Сохраняется:** В поле `photos` таблицы `cars` как JSON-массив
- **Формат файла:** `car_{telegram_id}_{timestamp}_{номер}.jpg`

## 📁 Структура файлов

```
uploads/
├── members/          # Фото участников
│   └── member_123456789_1703123456789.jpg
└── cars/            # Фото автомобилей
    ├── car_123456789_1703123456789_1.jpg
    ├── car_123456789_1703123456789_2.jpg
    └── car_123456789_1703123456789_3.jpg
```

## 🔧 Технические детails

### Telegram ID
- **Автоматически записывается:** Да! 
- **Источник:** `msg.from.id` из Telegram API
- **Поле в БД:** `telegram_id` (BIGINT)
- **Уникальность:** Да, каждый пользователь имеет уникальный ID

### Обработка фотографий
1. Бот получает массив размеров фото от Telegram
2. Выбирается самое большое разрешение (`msg.photo[msg.photo.length - 1]`)
3. Загружается через Telegram Bot API
4. Сохраняется локально в папку `uploads/`
5. Путь записывается в базу данных

### Команды для работы с фото
- `/skip` - пропустить загрузку фото
- `/done` - завершить загрузку фотографий автомобиля

## 🎯 Процесс регистрации с фото

1. `/register` - начало регистрации
2. Имя → Фамилия → Город → Телефон → О себе
3. **📸 Загрузка фото профиля** (новый шаг!)
4. Сохранение в БД с фотографией

## 🚗 Процесс добавления авто с фото

1. "Добавить авто" из меню
2. Марка → Модель → Год
3. **🔢 Регистрационный номер** (новый шаг!)
4. **📸 Загрузка фотографий** (новый шаг!)
5. Можно загрузить несколько фото подряд
6. `/done` для завершения
7. Сохранение в БД с массивом фотографий

### 🔢 Регистрационный номер
- **Формат:** Только латинские буквы (A-Z) и цифры (0-9)
- **Длина:** От 4 до 12 символов
- **Без пробелов и символов:** A123BC77, 1234AB199, H001AA
- **Можно пропустить:** Да, командой `/skip`
- **Автоматическое преобразование:** Приводится к верхнему регистру

## 📋 Системные требования

- **Node.js:** версия 18+ (для встроенного fetch API)
- **Текущая версия:** Node.js v22.11.0 ✅

Никаких дополнительных зависимостей для работы с фотографиями не требуется - используется встроенный `fetch`!
