- Использование `fs.createReadStream()` для локальных файлов
- Добавлена опция `filepath: false` в конфигурацию бота
- Улучшенное логирование процесса отправки фотографий
- Graceful fallback на текстовые уведомления при ошибках

### Детали исправления
```javascript
// Было:
await bot.sendPhoto(config.CHAT_ID, photoPath, options);

// Стало:
let photoSource = photoPath.startsWith('http') 
    ? photoPath 
    : fs.createReadStream(photoPath);
await bot.sendPhoto(config.CHAT_ID, photoSource, options);
```

## 🔧 Исправление системы приглашений

### Проблема
Система проверяла дубликаты только среди приглашений, позволяя участникам оставлять визитки у автомобилей других участников клуба.

### Решение
- Изменена логика проверки всех автомобилей в базе
- Добавлен метод `getMemberById()` в database.js
- При обнаружении автомобиля участника показывается информация о владельце
- Сохранена проверка дубликатов для приглашений

## 📸 Исправление загрузки фотографий

### Проблема
Пользователи могли загружать фото бесконечно без возможности завершить процесс.

- Добавлена команда `/done` для завершения загрузки фотографий
- Обновлена логика в этапе `photos` функции `handleCreateInvitation()`
- Исправлены сообщения: `/skip` - пропустить все, `/done` - завершить после минимум одного фото
- Убраны неуместные сообщения после загрузки
